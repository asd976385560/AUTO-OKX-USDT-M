# 风控硬上限详细说明

## 概述

以下为**绝对天花板**。触碰任一条 → 立即停手 + 进入 PAUSE（需主人手动恢复）。
**小灵也不可违反**——这是底线，不是建议。

## 硬上限表

| 限制项 | 默认推荐档 | 契约硬上限 | 说明 |
|--------|-----------|-----------|------|
| 单笔仓位占净值 | ≤ **5%** | ≤ **10%** | 开仓时计算保证金占比 |
| 杠杆（BTC/ETH） | **10x** | ≤ **10x** | 主流币可用较高杠杆 |
| 杠杆（山寨） | **5x** | ≤ **10x** | 山寨币默认更保守 |
| 同侧暴露 | ≤ 50% | ≤ **60%** | 同方向所有仓位保证金总和 |
| 并发持仓 | ≤ 3 仓 | ≤ **6 仓** | 同时持有的仓位数量 |
| 单次开仓保证金 | — | ≤ **10% 净值** | 独立于单笔仓位限制，额外强制 |

## 10% 净值硬上限计算

```
保证金预算 = 账户净值(totalEq) × 10%
每张合约保证金 = 标记价格(markPx) × 合约面值(ctVal) ÷ 杠杆(leverage)
最大张数 = floor(保证金预算 ÷ 每张合约保证金)
最终张数 = min(原计算张数, 最大张数)
```

### 计算示例（净值 $1,221 USDT，10x 杠杆）

| 币种 | markPx | ctVal | 每张保证金 | 10% 预算 $122 | 最大张数 |
|------|--------|-------|-----------|-------------|---------|
| BTC | $81,000 | 0.01 BTC | $810 | ❌ 超预算 | 0（需降杠杆） |
| BTC | $81,000 | 0.01 BTC | $81 @100x | $122 | 1（10x 不行，需更高杠杆） |
| ETH | $2,331 | 0.01 ETH | $23.3 | $122 | 5 |
| SOL | $95 | 0.1 SOL | $9.5 | $122 | 12 |
| DOGE | $0.11 | 1000 DOGE | $11 | $122 | 11 |

### 常见 Bug：勿用 ctVal 直接比较

```
❌ 错误：ctVal = 0.01 BTC = $810 > $122 → 排除
✅ 正确：每张保证金 = $810 ÷ 10 = $81 < $122 → 可以开 1 张

❌ 错误：DOGE ctVal = 1000 × $0.11 = $110 > $122 → 排除
✅ 正确：每张保证金 = $110 ÷ 10 = $11 < $122 → 可以开 11 张
```

## PAUSE 触发条件

| 条件 | 处理 |
|------|------|
| 单笔超 6% 净值 | 立即 PAUSE |
| 杠杆 > 10x | 立即 PAUSE |
| 同侧暴露 > 60% | 立即 PAUSE |
| 并发持仓 > 6 仓 | 立即 PAUSE |
| HTTP 401 / 签名失败 | 立即 PAUSE |
| 连续 3 轮开仓失败 | 立即 PAUSE |

PAUSE 后需要主人手动恢复。Job B 不得自动清除 `pause_reason`。

## 数据降级规则

| 新鲜度 | 处理 |
|--------|------|
| Job A 最近成功 ≤ 15 min | 正常决策 |
| 15-60 min | 降级模式：dim4 cap 6，禁止激进加仓 |
| > 60 min | 禁止新开仓，仅允许 CLOSE/IDLE/PAUSE |
